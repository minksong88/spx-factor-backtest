# 국면별 ETF 롱숏 스코어 맵

> **마지막 갱신: 2026-07-23** (갱신 시 이 파일과 아래 CSV들을 **덮어씀** — 날짜 파일 안 쌓음)
> 계산 코드: **`src/regime_asset_scores.py`** — `python src/regime_asset_scores.py` 로 재생성.

## 파일 구성 (국면별로 분리)
| 국면 | 파일 | 뜻 |
|---|---|---|
| **RF** | `regime_RF.csv` | 리플레이션 (실질↑ BEI↑) |
| **ST** | `regime_ST.csv` | 스태그·완화기대 (실질↓ BEI↑) |
| **RC** | `regime_RC.csv` | 침체·디스인플레 (실질↓ BEI↓) |
| **TG** | `regime_TG.csv` | 긴축·디스인플레 (실질↑ BEI↓) |

각 CSV 컬럼: `rank, ticker, beta, avg_ret_pct, n_blocks, quadrant, regime`. **β 내림차순 정렬**(rank 1 = 그 국면 최강 롱). 엑셀/뷰어에서 오름·내림 자유 정렬.

## 읽는 법 (중요)
- **β 부호 = 방향**: (+)면 그 국면 강세(롱 후보), (−)면 약세(숏 후보). β = 실질금리 1%p 움직일 때 자산 % 반응 = mean(블록수익 / |Δ실질|), 오라클(전체표본).
- **β 크기 비교는 "같은 국면(=같은 파일) 안에서만"** 의미 있음. 국면끼리 β 절대값 비교 금지(국면마다 Δ실질 스케일이 달라 분모가 다름 — ST가 β가 큰 건 그 국면 실질변동폭이 작아서지 "제일 세다"가 아님).
- **avg% = 그 국면 평균 매수후보유 수익률**(방향·체감용). β와 부호 갈리면 β 우선.
- **n_blocks 작으면(≤7) 신뢰도 낮음** (BLOK/BITO/BETZ/ARTY/QTUM/FNGS/SRVR 등 상장 짧은 종목).
- **직전 상대수익(리버설/모멘텀) 필터는 붙이지 말 것.** 검증 결론: 실시간 국면판정 위에선 포지셔닝 오버레이가 위험조정수익을 못 올림(국면오판 손실 증폭). 이 맵은 "국면을 사람이 정하고 쓰는 커닝페이퍼".

## 국면별 요약 (롱 top / 숏 bottom)
| 국면 | 대표 롱 (β 상위) | 대표 숏 (β 하위) |
|---|---|---|
| **RF** 리플레이션 | USO · KRE · KBE · FCG (에너지·은행) | GDX · SIL · SLV (금·귀금속) |
| **ST** 스태그·완화기대 | BLOK · COPX · SIL · PBW · GDX (실물·귀금속·성장) | UNG (거의 다 β+, 순수숏 드묾) |
| **RC** 침체·디스인플레 | GDX · REZ · SRVR · XLRE (금·리츠·유틸) | OIH · FCG · XLE (에너지·원자재) |
| **TG** 긴축·디스인플레 | SOXX · IHE · XLV · XBI (헬스·방산·귀금속) | BITO · BLOK · BETZ (크립토·고멀티플) |

## 국면 관통 패턴 (참고)
- **금·귀금속(GDX/SIL/SLV/GLD)**: RF 최악 → ST·RC 최고. 실질금리 역행 자산.
- **에너지·원자재(USO/FCG/OIH/COPX/XLE)**: RF 최고 → RC·TG 최악. 리플레이션 베타.
- **은행(KRE/KBE)**: RF 강세(금리 스팁), 그 외 중립~약세.
- **크립토·고멀티플(BITO/BLOK/BETZ)**: TG 최약(고 duration).
- **방어·헬스케어(XLV/IHE/XLP)**: 국면 무관에 가까움, TG서만 상대강세.
